4 Track Detection by Song Recognition
How Song Recognition Works

A short audio sample (~15 seconds) from the middle of each track is sent to the selected service. If the song is recognised, its title, artist, album and year are filled in automatically.

AudD — sign up for a free key at dashboard.audd.io ↗.

Shazam via RapidAPI — sign up at RapidAPI ↗ and subscribe to the Shazam API. Uses an unofficial endpoint.

Your API key is never stored — it lives only in the page for the duration of your session. Use Identify All/Selected Untagged to process every untagged track, or tick checkboxes in the Review card to target specific tracks.

Detect & Apply BPM to Track Metadata
Uses Essentia.js (RhythmExtractor2013) — more accurate than the detection algorithm estimate. Loads ~220 KB from CDN on first use. BPM is written to each track's metadata and editable in the Review panel.
5 Export Tracks
Export Options

FFmpeg Script .sh (macOS/Linux): Place in the same folder as your audio file, run chmod +x script.sh && ./script.sh. Produces fully-tagged MP3s. Get FFmpeg ↗

FFmpeg Script .bat (Windows): Place in the same folder as your audio file, then double-click or run from a command prompt. Requires FFmpeg on your PATH.

WAV (Lossless): Exports each track as a WAV file. Use Mp3tag or Kid3 to add ID3 tags afterwards.

Cue Sheet: A single .cue file describing all split points — use with your audio player or ripper.

Metadata JSON: A structured JSON file with all track metadata for use in custom workflows.
